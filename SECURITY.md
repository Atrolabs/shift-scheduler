# Security Policy

**Found something sketchy? Let us know — just don't blast it in a public issue.**

</div>

---

## Reporting a Vulnerability

Use [GitHub's private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) or reach out to the maintainers directly.

### What to include

| Detail | Why |
|--------|-----|
| Description | What's broken |
| Steps to reproduce | So we can see it ourselves |
| Potential impact | How bad is it |
| Suggested fix | If you've got one, even better |

### Response timeline

| Step | Timeframe |
|------|-----------|
| Acknowledgment | Within 48 hours |
| Initial assessment | Within 1 week |
| Fix or mitigation | ASAP, depending on severity |

## Supported Versions

Only the latest version on `master` is actively maintained.
